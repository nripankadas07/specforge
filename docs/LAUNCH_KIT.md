# SpecForge Launch Kit

Last updated: June 13, 2026.

SpecForge is designed to be easy to understand from a GitHub profile, but stars
come from distribution plus real utility. This kit keeps launch copy focused,
honest, and specific.

## Positioning

SpecForge turns high-signal open-source project patterns into a spec-driven
portfolio build plan. It is a local-first React/TypeScript app with a fixed
evidence dataset, transparent scoring, deterministic workflow simulation, and
Markdown blueprint export.

## Best Audience

| Audience | Why It Might Matter |
|---|---|
| Developers building portfolio projects | Helps choose a project direction with evidence instead of vibes. |
| AI workflow builders | Shows a small, inspectable spec-first workflow UI without requiring an LLM API. |
| Open-source maintainers | Provides a model for turning research signals into roadmap and README material. |
| Recruiters or reviewers | Gives a polished demo plus inspectable TypeScript logic and tests. |

## Demo Script

1. Open https://nripankadas07.github.io/specforge/.
2. Select three inspiration repositories with different strengths.
3. Toggle evidence depth and guardrails.
4. Inspect the ship score, risk, workflow graph, and event timeline.
5. Export the generated blueprint.
6. Open the source and inspect `src/lib/scoring.ts` and `src/lib/workflow.ts`.

## Launch Drafts

### Short Post

Built SpecForge: a local-first workflow studio that turns high-star OSS patterns
into a spec-driven project blueprint.

It includes a transparent scoring model, workflow graph, risk/confidence
signals, and Markdown export. No API key, no runtime scraping.

Demo: https://nripankadas07.github.io/specforge/
Source: https://github.com/nripankadas07/specforge

### Technical Post

I built SpecForge because choosing a strong GitHub-profile project is usually
too hand-wavy. The app starts from a fixed, source-linked dataset of high-star
developer projects, scores inspirations across social proof, technical depth,
demo appeal, feasibility, moat, and risk, then turns that into a deterministic
spec-first workflow.

The goal is not to predict popularity. The goal is to make project selection
inspectable enough that another developer can disagree with the weights, change
the data, and still understand the system.

Demo: https://nripankadas07.github.io/specforge/
Source: https://github.com/nripankadas07/specforge

## Feedback Requests

Ask for narrow feedback so comments can become useful issues or pull requests:

- Are the scoring dimensions clear enough?
- Does the generated blueprint help someone start a real project?
- Which workflow node should be more detailed?
- Should the dataset remain fixed, support local imports, or add cached live
  GitHub fetching?
- What would make this useful for teams, not only individual portfolio work?

## Do Not Claim

- Do not claim SpecForge predicts star growth.
- Do not claim the dataset is live or exhaustive.
- Do not claim it uses all social platforms at runtime.
- Do not claim LLM-generated project advice.
- Do not imply external users or adoption until those exist.

The strongest pitch is the real one: SpecForge is a polished, local-first,
inspectable product demo for evidence-based project planning.
